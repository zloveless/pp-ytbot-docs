What's here

This is the source code for the ytbot viewer docs — the website that explains the points system and lists all the chat commands for the stream. It's a static site built with Astro.

The files you'll touch most often when something changes:

What you want to changeFile to edit
Add, remove, or edit a command src/data/commands/vanilla-early.json
Add or rename a player target (@handle) src/data/targets.json
Change the "How It Works" explanations src/pages/how-it-works.md
Change the home page blurb src/pages/index.md
Change the spawn mechanics page src/pages/commands/spawns.mdx
Tip: You never need to touch anything in src/components/, src/layouts/, or src/styles/ for routine content updates.

Prerequisites

You only need to do this once on a new machine.

1

Install Node.js (v22 or newer). Download from nodejs.org — grab the LTS version.

2

Enable Corepack (this gives you Yarn):

corepack enable
3

Clone the repo (if you haven't already):

git clone https://github.com/pseudoposse/ytbot_docs.git
cd ytbot_docs

Local setup

Run these two commands from the repo folder every time you start working:

1

Install dependencies (only needed after cloning or when packages change):

yarn install
2

Start the preview server:

yarn dev

Open http://localhost:4321 in your browser. The page live-reloads as you save files — no need to restart the server.

Note: Always use yarn, never npm or npx. This project uses Yarn 4 and will behave unexpectedly if you mix package managers.

Updating commands

All commands live in src/data/commands/vanilla-early.json. It's a list of objects — one object per command. Open the file in any text editor.

Adding a new command

Copy an existing entry and add it to the list. Every entry needs these fields:

FieldWhat it isExample
keywordThe chat trigger word (no !)"burn"
categoryOne of: negative, positive, funny, spawn"negative"
effectOne-line description shown in the table"Burns target for 5 seconds"
costPoint cost (number, no quotes)100
targetOne of: player, self, all"player"
versionAlways "vanilla" for now"vanilla"
stageAlways "early" for now"early"

Optional fields:

FieldWhat it isExample
superChatMinMinimum Super Chat in USD to trigger this command2.00
notesAny extra info shown in the Notes column"Stacks with burn"
baseCountSpawn commands only — minimum zombie count3
extraZombieThresholdSpawn commands only — viewers per extra zombie10

Example entry:

{
  "keyword": "burn",
  "category": "negative",
  "effect": "Burns target for 5 seconds",
  "cost": 100,
  "target": "player",
  "superChatMin": 1.00,
  "version": "vanilla",
  "stage": "early"
}
Watch the commas: JSON is picky. Every entry in the list needs a comma after its closing } — except the very last one. If the preview server throws an error, a missing or extra comma is the most likely cause.

Editing an existing command

Find the entry by its keyword and change whatever field you need. Save the file and the preview reloads automatically.

Removing a command

Delete the entire { … } block for that command, including the trailing comma on the entry before it.

Updating valid targets

Valid targets (the @handles viewers can type) are in src/data/targets.json. This single file controls both the Valid Targets table on the Commands page and anywhere else targets are referenced.

Each target has three fields:

FieldWhat it isExample
handleThe @mention without the @"eerie"
typeEither player (a streamer) or special (a bot keyword)"player"
noteExplanation shown in the Notes column. Use null if there's nothing to add.null

Renaming a player

Find their entry and change the handle value. Save — done.

Adding a new player

Add a new entry to the list:

{ "handle": "newperson", "type": "player", "note": null }

Removing a player

Delete their { … } entry from the list (watch the commas).

Publishing changes

The site deploys automatically via GitHub Actions whenever you push to the main branch. There is no manual deploy step — GitHub builds and publishes the site to GitHub Pages for you.

1

Preview your changes locally with yarn dev and check that everything looks right in the browser.

2

Stage and commit your changes:

git add src/data/commands/vanilla-early.json
git commit -m "Add new command: freeze"

Replace the file path and message with whatever you actually changed.

3

Push to GitHub:

git push

GitHub Actions will build and publish the site within a minute or two. You can watch the progress under the Actions tab in the GitHub repo.

Want to double-check before publishing? Run yarn build locally — if it finishes without errors, the deploy will succeed. The built files land in dist/ but you don't need to commit them.
One-time repo setup: After forking or transferring this repo, go to Settings → Pages in GitHub and set the source to "GitHub Actions". You only need to do this once — after that, every push to main deploys automatically.
